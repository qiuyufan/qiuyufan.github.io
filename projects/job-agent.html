AI Job Agent

A personal project that transforms the chaotic job search process into a structured, data-driven workflow using AI-assisted automation and analytics.

  • Python
  • LLM Workflow
  • Streamlit
  • SQLite
  • Automation
  • Analytics

Why I Built This

When I started looking for a new job, I realized the biggest challenge wasn't finding opportunities — it was managing the entire process.

Jobs were scattered across LinkedIn, Glassdoor, company career pages, and recruitment websites. Notifications quickly became spam, and whenever work got busy, I would stop checking for a while and sometimes discover that a perfect position had already passed its application deadline.

Even when I found an interesting job, the process was repetitive. I had to locate my latest CV, copy the job description into ChatGPT, generate a cover letter, and somehow remember what I had already applied for.

After a month I couldn't answer simple questions such as:

  • How many jobs have I applied for this month?
  • Which websites actually lead to interviews?
  • Why did I reject certain opportunities?
  • Which applications are still active?

As someone working in analytics, my instinct was to treat the job search as a data problem rather than a manual process.

From Problems to Solutions

ProblemMy Solution
Too many job platforms. I constantly worried about missing good opportunities across multiple websites. Built one centralized dashboard that collects jobs into a single searchable workspace.
Notification overload. Job alerts quickly became background noise. Replaced email notifications with one dashboard that I review on my own schedule.
No structured workflow. Some weeks I actively applied for jobs; other weeks I became busy and completely forgot. Designed an application pipeline with clear stages (Interested → Applied → Interview → Offer → Rejected).
Repetitive application process. Every application involved repeating the same manual steps. Kept job descriptions, notes, AI-assisted analysis, and application history together in one place.
Hard to prioritize. Reading every job description carefully was time-consuming. Used AI to extract key requirements, compare them with my CV, and provide a recommendation to support my decision.
No way to learn from the process. I couldn't see patterns in my applications. Built analytics showing application funnel, source quality, recommendation levels, and overall progress.

Key Design Decisions

Rather than simply listing features, I wanted every feature to solve a specific problem.

FeatureWhy I Built It
Centralized application pipeline To replace scattered spreadsheets, browser tabs, and emails with one workspace.
AI-assisted job evaluation To reduce the time spent reading long job descriptions while still making informed decisions.
Search To quickly remember previous applications and avoid duplicate work.
Analytics dashboard To understand which sources and strategies were actually producing results.
Status tracking To ensure every opportunity had a clear next step.

What I Learned

  • AI becomes valuable after the workflow is structured. Initially I thought AI would be the most valuable part of the project. After using the application daily, I realized the biggest improvement came from organizing information first. Once the workflow was structured, AI naturally became much more useful.
  • Building products is different from building features. The hardest part wasn't implementing functionality — it was deciding which problems were actually worth solving and keeping the workflow simple enough that I would continue using it every day.
  • Dashboards should support decisions. A dashboard isn't successful because it contains many charts. It's successful if it helps users make faster and better decisions with less effort.
  • AI accelerated development, but engineering decisions were still mine. I used AI extensively during development, but I was responsible for defining the workflow, evaluating the outputs, refining the product, and deciding where automation genuinely added value.

If I Built It Again Today

Looking back, I would spend less time adding features and more time improving the decision-making workflow. Some improvements I'd like to explore include:

  • Only run AI evaluation after simple filters identify promising opportunities.
  • Learn from my application history to improve future recommendations.
  • Measure interview rates, response rates, and source quality over time.
  • Create a synthetic demo dataset so the project can be demonstrated publicly without exposing personal application data.
  • Deepen my understanding of the AI-generated architecture so I can extend and maintain the project more confidently.
